Descrizione dettagliata

Implementazione concreta dell'interfaccia Track.

  • Questa classe rappresenta fisicamente una traccia musicale nel sistema. Gestisce lo stato interno dei dati e applica le regole di validazione al momento dell'inserimento o della modifica.

◆ equals()

boolean com.example.gruppo04.model.TrackImpl.equals ( Object o)

Confronta questa traccia con un altro oggetto per verificarne l'uguaglianza.

  • L'uguaglianza tra due tracce deve essere basata esclusivamente sul confronto dei loro identificatori univoci (UUID), ignorando del tutto campi come titolo o autore.
  • Parametri
    oL'oggetto da confrontare con la traccia corrente.
    Restituisce
    true se gli oggetti hanno lo stesso UUID, false altrimenti.

◆ getFilePath()

String com.example.gruppo04.model.TrackImpl.getFilePath ( )

Recupera il percorso fisico del file audio associato alla traccia.

  • Questo metodo restituisce lo stato interno della variabile filePath, che rappresenta l'esatta locazione del file nel sistema operativo. È essenziale per il passaggio del file al motore del MediaPlayer.
  • Restituisce
    Il percorso assoluto come stringa, oppure null se la traccia è stata creata solo testualmente senza file associato.
