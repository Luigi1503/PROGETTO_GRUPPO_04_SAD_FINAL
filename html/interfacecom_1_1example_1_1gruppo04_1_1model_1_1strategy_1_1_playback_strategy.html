Descrizione dettagliata

Interfaccia per la strategia di riproduzione dei brani musicali.

Definisce il contratto per la selezione della sorgente successiva all'interno di una lista di sorgenti riproducibili, in base alla strategia scelta.

◆ nextTrack()

Track com.example.gruppo04.model.strategy.PlaybackStrategy.nextTrack ( List< Track > tracks,
int currentIndex )

Seleziona la traccia successiva all'interno di una sorgente.

La logica varia in base alla strategia:

  • Sequential: traccia successiva in ordine; null se è l'ultima.
  • Loop: traccia successiva in ordine; torna alla prima se è l'ultima.
  • Shuffle: traccia casuale tra quelle disponibili, diversa dalla corrente.
Parametri
tracksla lista di Track della sorgente corrente.
currentIndexl'indice della traccia corrente nella lista.
Restituisce
la traccia successiva selezionata, o null se la sorgente è terminata e si deve passare alla sorgente successiva.

Implementato in com.example.gruppo04.model.strategy.LoopStrategy, com.example.gruppo04.model.strategy.SequentialStrategy, e com.example.gruppo04.model.strategy.ShuffleStrategy.

PlayableSource com.example.gruppo04.model.strategy.PlaybackStrategy.previousSource ( List< PlayableSource > sources,
int currentIndex )

Seleziona la sorgente precedente nella lista.

È l'operazione simmetrica a nextSource(List, int) e ne rispecchia la logica per ciascuna strategia:

  • Sequential: sorgente precedente in ordine; null se è la prima.
  • Shuffle: una sorgente casuale ("playlist casuale dal catalogo").
  • Loop: resta sulla sorgente corrente (la playlist riparte da capo).
Parametri
sourcesla lista di PlayableSource disponibili.
currentIndexl'indice della sorgente corrente nella lista.
Restituisce
la sorgente precedente selezionata, o null se non esiste.

◆ previousTrack()

Track com.example.gruppo04.model.strategy.PlaybackStrategy.previousTrack ( List< Track > tracks,
int currentIndex )

Seleziona la traccia precedente all'interno di una sorgente.

È l'operazione simmetrica a nextTrack(List, int) e ne rispecchia la logica per ciascuna strategia:

  • Sequential: traccia precedente in ordine; null se è la prima (segnale per passare alla sorgente precedente).
  • Loop: traccia precedente in ordine in modo ciclico; dalla prima torna all'ultima (con un'unica traccia resta sulla stessa).
  • Shuffle: traccia precedentemente riprodotta (dallo storico); se non esiste, la traccia corrente (così da riavviarla da capo).
Parametri
tracksla lista di Track della sorgente corrente.
currentIndexl'indice della traccia corrente nella lista.
Restituisce
la traccia precedente selezionata, o null se si è all'inizio della sorgente e si deve passare alla sorgente precedente.

Implementato in com.example.gruppo04.model.strategy.LoopStrategy, com.example.gruppo04.model.strategy.SequentialStrategy, e com.example.gruppo04.model.strategy.ShuffleStrategy.

◆ usesHistoryForPrevious()

default boolean com.example.gruppo04.model.strategy.PlaybackStrategy.usesHistoryForPrevious ( )

Indica se la navigazione "indietro" a livello di traccia (⏮) deve basarsi sullo storico di riproduzione effettivo anziché su previousTrack.

Per le strategie deterministiche (Sequential, Loop) il brano precedente è ricavabile dall'ordine, quindi il valore è false. Per lo Shuffle il "brano precedentemente riprodotto" può trovarsi in un'altra sorgente (es. nel catalogo, dove ogni traccia è una sorgente a sé): solo lo storico reale, mantenuto dal client, può individuarlo, quindi il valore è true.

Restituisce
true se ⏮ deve usare lo storico di riproduzione del client.

Implementato in com.example.gruppo04.model.strategy.ShuffleStrategy.
